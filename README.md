Rack Coach – Project Charter v1.0

Project Name

Rack Coach

Alpha Team: The Assylum

Captain: David Goggans

Co-Captain: Kelley Goggans

---

Vision

Create an offline-first coaching application that helps APA captains make better decisions before, during, and after league matches.

The application should replace a traditional paper binder by providing instant access to player information, team strategy, lineup planning, and match history while remaining simple enough to use during live league play.

The ultimate objective is to help captains develop players more effectively and maximize team performance through data-driven coaching.

---

Mission Statement

Help captains deliver high-quality team strategies on match night while providing the best practice drills for each player based on their APA Skill Level, performance trends, and coaching needs.

---

Initial Target Users

Phase 1

The Assylum

This project will be developed around the real needs of The Assylum throughout an active APA session.

Every feature must provide value during an actual league night.

---

Future Users

Although Phase 1 is designed exclusively for The Assylum, the software architecture should support expansion to:

- Any APA 9-Ball team
- APA 8-Ball teams
- League operators (future)
- Tournament directors (future)

The application should be designed from the beginning to support multiple teams without requiring significant redesign.

---

Core Features (Version 1.0)

Team Management

- Team roster
- Player profiles
- Skill Levels
- Active/inactive status
- Contact information
- Coaching notes

---

Match Night

- Create new match
- Opponent tracking
- Match history
- Player selection
- Match notes
- Defensive shot tracking
- Timeout tracking

---

23-Rule Calculator

- Live total calculation
- Visual validation
- Suggested legal lineups
- Availability filtering

---

Statistics

Track:

- Wins
- Losses
- Win Percentage
- Points Per Match (PPM)
- Percentage of Available Points (PA%)
- Defensive shots
- Timeouts
- Match history

---

Player Development

Each player should have:

- Strengths
- Weaknesses
- Coaching notes
- Practice assignments
- Drill history
- Improvement goals

---

Offline First

The application must function without internet access.

All data will be stored locally.

Users may optionally export backups.

Internet access should never be required during league play.

---

Non-Goals (Phase 1)

The following features are intentionally deferred:

- AI coaching
- Cloud synchronization
- League website integration
- Video analysis
- Tournament management
- Multiple organizations

These features will be considered after a stable Version 1.0.

---

Success Criteria

The project will be considered successful when:

- The captain can complete an entire league night using only Rack Coach.
- The application replaces the physical captain's binder.
- Lineups can be built using the built-in 23-rule calculator.
- Match data can be entered quickly during play.
- Player development plans are available at any time.
- A coach evaluates the application and provides structured feedback for future improvements.

---

Design Principles

- Offline-first
- Mobile-first
- Fast and responsive
- Easy to learn
- Minimal taps
- Large touch targets
- Dark mode optimized
- No unnecessary complexity

---

Long-Term Vision

Rack Coach should become the standard coaching platform for APA captains by combining player development, match strategy, statistics, and coaching knowledge into a single application.

Every new feature must answer one question:

Does this help the captain make a better coaching decision?
